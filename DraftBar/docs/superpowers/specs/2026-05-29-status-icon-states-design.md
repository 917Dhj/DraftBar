# DraftBar 菜单栏图标状态设计规格

日期：2026-05-29

## 1. 背景

DraftBar 的核心交互是从菜单栏把一张 memo 编辑页拖到桌面上。当前菜单栏图标已经采用“空心便签 + 两条笔记线 + 实心笔 + 小 V”的 template 图标，用小 V 暗示用户可以向下拖动。

产品状态可以简化为两类：

1. 编辑页尚未显示在桌面上。
2. 编辑页已经显示在桌面上。

这两个状态应使用相近但不同的菜单栏图标，帮助用户理解当前状态和下一步操作。

## 2. 目标

设计两种菜单栏 template 图标，并定义切换行为：

- 未拖出状态：提示用户可以向下拖出 memo。
- 已显示状态：提示 memo 已经在桌面上，点击菜单栏图标会聚焦编辑页。

图标状态应保持克制，不把菜单栏图标变成复杂控件。两种状态必须在 18px 菜单栏尺寸下仍然可读。

## 3. 非目标

本阶段不做以下内容：

- 不把菜单栏图标做成动画。
- 不在图标中加入颜色、透明度变化或阴影。
- 不使用徽章、圆点、对勾、上箭头等额外状态符号。
- 不改变页面内已有关闭按钮的职责。
- 不让菜单栏图标承担“收回/隐藏窗口”的主操作。

## 4. 图标语义

### 4.1 未拖出状态

图标元素：

- 空心圆角便签轮廓。
- 两条短笔记线。
- 右下角实心斜笔。
- 便签下方短小圆角 V。

语义：

- 便签 + 笔表示这是一个可写 memo。
- 小 V 表示 memo 可以向下拖出。

这个状态适用于 app 启动后、编辑页关闭后、或没有可见 memo 编辑页时。

### 4.2 已显示状态

图标元素：

- 空心圆角便签轮廓。
- 两条短笔记线。
- 右下角实心斜笔。
- 不包含小 V。

语义：

- memo 已经显示在桌面上。
- 点击菜单栏图标应聚焦当前编辑页，而不是收回或关闭。

已显示状态不使用小上箭头。原因是上箭头会自然暗示“收回/隐藏”，而产品里已经有页面内关闭按钮，菜单栏图标点击更适合作为聚焦入口。

## 5. 状态切换

状态来源应以编辑页 panel 的可见性为准。

切换规则：

1. app 启动且编辑页未显示：使用未拖出图标。
2. 用户点击菜单栏图标打开编辑页：切换为已显示图标。
3. 用户从菜单栏向下拖出编辑页：开始显示或拖出成功后切换为已显示图标。
4. 编辑页已经显示时，点击菜单栏图标：聚焦现有编辑页，图标保持已显示状态。
5. 用户点击页面内关闭按钮，关闭动画完成后：切回未拖出图标。

如果关闭动画正在进行，图标状态应在实际 `orderOut` 后切回未拖出，避免图标先于窗口状态变化。

## 6. 工程设计

资产建议：

- `StatusBarIconHidden`：未拖出状态，包含小 V。
- `StatusBarIconVisible`：已显示状态，不包含小 V。

两个资产都应是 template image，并保持同一套线宽、笔形、便签尺寸和视觉重心。两者差异应主要集中在小 V 是否存在，以及已显示状态下可根据需要轻微居中便签。

代码建议：

- 在 status item 配置阶段加载未拖出图标作为默认图标。
- 增加一个小型 helper，例如 `updateStatusItemIcon(isNoteVisible:)`。
- `showFloatingNote`、`showFloatingNoteForDrag`、`closeFloatingNote` 的完成路径负责调用这个 helper。
- 右键菜单只显示菜单，不应改变图标状态。

## 7. 验收标准

视觉验收：

1. 未拖出状态在 18px 下能读出便签、笔和小 V。
2. 已显示状态在 18px 下能读出便签和笔，且没有小 V。
3. 两种状态并排看时属于同一套图标系统。
4. 已显示状态不暗示收回、下载或展开菜单。

交互验收：

1. app 启动时显示未拖出图标。
2. 打开或拖出编辑页后切换为已显示图标。
3. 编辑页显示时点击菜单栏图标会聚焦编辑页，不关闭窗口。
4. 页面内关闭按钮关闭编辑页后切回未拖出图标。
5. 右键菜单不会错误改变图标状态。

工程验收：

1. 两套状态图标都在 asset catalog 中声明为 template。
2. 图标切换不影响当前 status item 点击、右键菜单和拖拽逻辑。
3. Xcode 构建通过。
4. 资产校验覆盖两套状态图标的存在、尺寸和 template 设置。
